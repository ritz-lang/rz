## What I Learned from Prism's Conversation

The key insights that will affect Angelo's design:

1. **Prism uses a hybrid rendering model** — Command mode (X11-like) for simple things like text, buffer mode (Wayland-like) for complex rendering. Angelo needs to support both efficiently.

2. **Remote-friendly protocol** — Commands are serializable for SSH tunneling. This means glyph rendering might happen on the server side (command mode) OR client side (buffer mode).

3. **Triple buffering + async scanline compositor** — Prism's compositor is sophisticated. Angelo's glyph cache needs to be thread-safe and fast.

4. **RenderCommand.DrawText** — This is how Prism will call Angelo:
   ```ritz
   DrawText { pos: Point, text: StrView, font: FontId, size: u32 }
   ```

---

## TTF Loader Architecture

The design is **zero-copy where possible** and **lazy loading**:

1. **On load**: Parse offset table, table directory, and essential tables (head, hhea, maxp)
2. **On first character lookup**: Parse cmap table
3. **On first glyph access**: Parse loca table (glyph offsets)
4. **On glyph render**: Parse individual glyph from glyf table

This means opening a font is fast, and we only do work when needed.
