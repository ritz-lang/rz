## 🏗️ Thinking Through MVRSPT - The Ritzy Way

You're right that **MVRSPT** is logical backend-to-frontend order:

```
Model → View → Repository → Service → Presenter → Tests
  ↓        ↓        ↓           ↓          ↓          ↓
 Data   Render   Storage   Business    HTTP     Verify
```

But the **flow of a request** is actually:

```
HTTP Request
    ↓
Presenter (route handler, parses request)
    ↓
Service (business logic, orchestrates)
    ↓
Repository (data access)
    ↓
Model (data structures)
    ↓
Repository returns Model
    ↓
Service applies rules
    ↓
Presenter selects View
    ↓
View (renders response)
    ↓
HTTP Response
```

### Key Insights for Ritz

1. **No runtime reflection** - We can't do magic Rails-style "convention over configuration" discovery at runtime. Everything needs to be explicit or generated at compile time.

2. **No inheritance** - We use composition + traits. The `Repository<T>` trait is the abstraction.

3. **No closures yet** - Middleware can't be arbitrary functions. We need struct-based middleware.

4. **Explicit is ritzy** - The language philosophy is "no magic" - direct syscalls, no libc, etc. The framework should match.

### Proposed Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│                         SPIRE FRAMEWORK                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐                    │
│   │  Model   │     │   View   │     │   Tests  │                    │
│   │ structs  │     │ templates│     │  mocks   │                    │
│   └──────────┘     └──────────┘     └──────────┘                    │
│        │                │                │                           │

│   ┌──────────────────────────────────────────────────────────┐      │
```

## 1. Models - Pure Data Structs

Models should be dead simple - just data, no behavior. Validation happens in Services.

```ritz
# models/task.ritz

import spire.model { Model, Field, Uuid, Timestamp }

# The @model attribute generates:
# - Serialization to/from JSON
# - Field metadata for migrations
# - Default constructor
@model
struct Task
    id: Uuid
    title: String
    description: Option<String>
    completed: bool
    created_at: Timestamp
    updated_at: Timestamp
    user_id: Uuid              # Foreign key (just data, not enforced here)

# Relationships are explicit, not magic
@model
struct User
    id: Uuid
    email: String
    name: String
    created_at: Timestamp
```

**Design decisions:**
- `@model` is compile-time, generates serialization code
- No ActiveRecord pattern - models don't know about the database
- Foreign keys are just fields - relationships are in Repository queries

- Type-safe (written in Ritz, not SQL strings)
- Reversible

## 3. Repository - Data Access Layer

Repository is the abstraction over storage. It's a trait so we can swap implementations (Mausoleum for production, Mock for tests).

```ritz
# repos/task_repo.ritz

import spire.repo { Repository, Query, QueryResult }
import mausoleum { Db }
import models.task { Task }
import models.user { User }

# The trait - defines the contract
trait TaskRepository
    fn find(self: *Self, id: Uuid) -> Option<Task>
    fn find_all(self: *Self) -> Vec<Task>
    fn find_by_user(self: *Self, user_id: Uuid) -> Vec<Task>
    fn find_pending(self: *Self, user_id: Uuid) -> Vec<Task>
    fn create(self: &Self, task: *Task) -> Result<Task, RepoError>
    fn update(self: &Self, task: *Task) -> Result<Task, RepoError>
    fn delete(self: &Self, id: Uuid) -> Result<(), RepoError>
    fn count_pending(self: *Self, user_id: Uuid) -> i64

# Production implementation - uses Mausoleum
struct MausoleumTaskRepo
    db: *Db

fn mausoleum_task_repo_new(db: *Db) -> MausoleumTaskRepo
    MausoleumTaskRepo { db: db }

impl TaskRepository for MausoleumTaskRepo
    fn find(self: *MausoleumTaskRepo, id: Uuid) -> Option<Task>
        self.db.query(c"tasks")
            .where_eq(c"id", &id)
            .first::<Task>()
    
    fn find_by_user(self: *MausoleumTaskRepo, user_id: Uuid) -> Vec<Task>
        self.db.query(c"tasks")
            .where_eq(c"user_id", &user_id)
            .order_by(c"created_at", Desc)
            .all::<Task>()
    
    fn find_pending(self: *MausoleumTaskRepo, user_id: Uuid) -> Vec<Task>
        self.db.query(c"tasks")
            .where_eq(c"user_id", &user_id)
            .where_eq(c"completed", false)
            .all::<Task>()
    
    fn create(self: &MausoleumTaskRepo, task: *Task) -> Result<Task, RepoError>
        task.id = uuid_new_v4()
        task.created_at = timestamp_now()
        task.updated_at = timestamp_now()
        self.db.insert(c"tasks", task)
    
    fn update(self: &MausoleumTaskRepo, task: *Task) -> Result<Task, RepoError>
        task.updated_at = timestamp_now()
        self.db.update(c"tasks", &task.id, task)
    
    fn delete(self: &MausoleumTaskRepo, id: Uuid) -> Result<(), RepoError>
        self.db.delete(c"tasks", &id)
    
    fn count_pending(self: *MausoleumTaskRepo, user_id: Uuid) -> i64
        self.db.query(c"tasks")
            .where_eq(c"user_id", &user_id)
            .where_eq(c"completed", false)
            .count()
```

## 4. Services - Business Logic

Services contain all business rules. They're testable because they take repositories as dependencies.

```ritz
# services/task_service.ritz

import spire.service { ServiceError, ValidationError, LimitError }
import repos.task_repo { TaskRepository }
import models.task { Task }
import models.user { User }

struct TaskService<R: TaskRepository>
    repo: *R
    max_pending_tasks: i64

fn task_service_new<R: TaskRepository>(repo: *R) -> TaskService<R>
    TaskService::<R> {
        repo: repo,
        max_pending_tasks: 100
    }

impl<R: TaskRepository> TaskService<R>
    # Create a task with business rule validation
    fn create_task(self: &TaskService<R>, user: *User, title: *String) -> Result<Task, ServiceError>
        # Rule: title must be meaningful
        if title.len() < 3
            return Err(ValidationError(c"Title must be at least 3 characters"))
        
        if title.len() > 200
            return Err(ValidationError(c"Title must be at most 200 characters"))
        
        # Rule: max pending tasks per user
        let pending: i64 = self.repo.count_pending(user.id)
        if pending >= self.max_pending_tasks
            return Err(LimitError(c"Too many pending tasks (max 100)"))
        
        # Create the task
        var task: Task = Task {
            id: uuid_zero(),           # Will be set by repo
            title: title.clone(),
            description: None,
            completed: false,
            created_at: timestamp_zero(),  # Will be set by repo
            updated_at: timestamp_zero(),
            user_id: user.id
        }
        
        self.repo.create(&task)
    
    fn complete_task(self: &TaskService<R>, user: *User, task_id: Uuid) -> Result<Task, ServiceError>
        # Find the task
        match self.repo.find(task_id)
            None => Err(NotFoundError(c"Task not found"))
            Some(task) =>
                # Rule: can only complete your own tasks
                if !uuid_eq(&task.user_id, &user.id)
                    return Err(ForbiddenError(c"Cannot modify another user's task"))
                
                # Rule: can't complete an already completed task
                if task.completed
                    return Err(ValidationError(c"Task is already completed"))
                
                task.completed = true
                self.repo.update(&task)
    
    fn list_tasks(self: *TaskService<R>, user: *User) -> Vec<Task>
        self.repo.find_by_user(user.id)
    
    fn get_task(self: *TaskService<R>, user: *User, task_id: Uuid) -> Result<Task, ServiceError>
        match self.repo.find(task_id)
            None => Err(NotFoundError(c"Task not found"))
            Some(task) =>
                if !uuid_eq(&task.user_id, &user.id)
                    Err(ForbiddenError(c"Cannot view another user's task"))
                else
                    Ok(task)
```

### Option B: Ritz Builder Pattern (programmatic)

```ritz
# views/tasks.ritz

import spire.view { Html, Element, el, text, attr }
import models.task { Task }

fn tasks_index(tasks: *Vec<Task>) -> Html
    layout_base(c"My Tasks",
        el(c"div").class(c"tasks").children(
            el(c"h1").text(c"My Tasks"),
            
            if tasks.is_empty()
                el(c"p").text(c"No tasks yet. Create one!")
            else
                el(c"ul").children_each(tasks, task_item),
            
            el(c"a").href(c"/tasks/new").class(c"btn").text(c"New Task")
        )

fn task_item(task: *Task) -> Element
    var classes: String = string_from(c"task")
    if task.completed
        classes.push_str(c" completed")
    
    el(c"li").class_str(&classes).children(
        el(c"a").href_fmt(c"/tasks/{}", &task.id).text_str(&task.title),
        
        if !task.completed
            el(c"form").method(c"POST").action_fmt(c"/tasks/{}/complete", &task.id).children(
                el(c"button").attr(c"type", c"submit").text(c"Complete")
            )
        else
            empty()
    )
```

**Cons:** Verbose, harder for designers, mixes layout with code

## 6. Presenters - HTTP Handlers

Presenters wire HTTP to Services and Views.

```ritz
# presenters/task_presenter.ritz

import spire.http { Request, Response, Router }
import spire.view { render }
import services.task_service { TaskService }
import views.tasks { tasks_index, tasks_show, tasks_new, tasks_edit }
import views.json { json_tasks, json_task }

struct TaskPresenter<R: TaskRepository>
    service: *TaskService<R>

fn task_presenter_new<R: TaskRepository>(service: *TaskService<R>) -> TaskPresenter<R>
    TaskPresenter::<R> { service: service }

impl<R: TaskRepository> TaskPresenter<R>
    # GET /tasks
    fn index(self: *TaskPresenter<R>, req: *Request) -> Response
        let user: *User = req.user()  # From auth middleware
        let tasks: Vec<Task> = self.service.list_tasks(user)
        
        # Content negotiation
        if req.wants_json()
            Response.ok().json(json_tasks(&tasks))
        else
            Response.ok().html(render(tasks_index(&tasks)))
    
    # GET /tasks/:id
    fn show(self: *TaskPresenter<R>, req: *Request) -> Response
        let user: *User = req.user()
        let id: Uuid = req.param_uuid(c"id")
        
        match self.service.get_task(user, id)
            Ok(task) =>
                if req.wants_json()
                    Response.ok().json(json_task(&task))
                else
                    Response.ok().html(render(tasks_show(&task)))
            Err(NotFoundError(_)) =>
                Response.not_found()
            Err(ForbiddenError(_)) =>
                Response.forbidden()
    
    # GET /tasks/new
    fn new_form(self: *TaskPresenter<R>, req: *Request) -> Response
        Response.ok().html(render(tasks_new()))
    
    # POST /tasks
    fn create(self: &TaskPresenter<R>, req: *Request) -> Response
        let user: *User = req.user()
        
        # Parse form or JSON body
        let title: String = req.form_string(c"title")
        
        match self.service.create_task(user, &title)
            Ok(task) =>
                if req.wants_json()
                    Response.created().json(json_task(&task))
                else
                    Response.redirect(fmt(c"/tasks/{}", task.id))
            Err(ValidationError(msg)) =>
                if req.wants_json()
                    Response.bad_request().json_error(msg)
                else
                    Response.ok().html(render(tasks_new_with_error(msg, &title)))
            Err(LimitError(msg)) =>
                Response.conflict().json_error(msg)
    
    # POST /tasks/:id/complete
    fn complete(self: &TaskPresenter<R>, req: *Request) -> Response
        let user: *User = req.user()
        let id: Uuid = req.param_uuid(c"id")
        
        match self.service.complete_task(user, id)
            Ok(task) =>
                if req.wants_json()
                    Response.ok().json(json_task(&task))
                else
                    Response.redirect(c"/tasks")
            Err(err) =>
                handle_service_error(req, err)

    router.get(c"/tasks", presenter, TaskPresenter::index)
    router.get(c"/tasks/new", presenter, TaskPresenter::new_form)
    router.get(c"/tasks/:id", presenter, TaskPresenter::show)
    router.post(c"/tasks", presenter, TaskPresenter::create)
    router.post(c"/tasks/:id/complete", presenter, TaskPresenter::complete)
```

### 1. Attributes - How `[[model]]` Works Generically

You're absolutely right - we don't want compiler changes for Spire-specific attributes. Here's how we make it generic:

```ritz
# The compiler just parses [[name(args)]] and stores it in AST
# The FRAMEWORK reads these at compile-time via ritzgen tooling

[[model]]                    # Compiler: stores Attribute("model", [])
[[route("GET", "/tasks")]]   # Compiler: stores Attribute("route", ["GET", "/tasks"])
[[packed]]                   # Compiler: stores Attribute("packed", [])

# Spire provides a code generator that reads these
# Similar to how ritzgen works for the parser

# spire_gen.py (or ritzgen plugin)
# 1. Parses .ritz files
# 2. Finds [[model]] structs
# 3. Generates serialization code
# 4. Generates migration metadata
```

**The key insight:** The compiler is dumb about attributes (just stores them). Code generators are smart about what to do with them.


### 3. Repository - Ownership Done Right

You're absolutely right about the pointer mess. Let's fix it with RERITZ:

```ritz
# repos/task_repo.ritz

import spire.repo { Repository, Query }
import mausoleum { Db }
import models.task { Task }

# Trait with proper RERITZ signatures
trait TaskRepository
    fn find(self, id: Uuid) -> Option<Task>        # const borrow of self
    fn find_all(self) -> Vec<Task>
    fn find_by_user(self, user_id: Uuid) -> Vec<Task>
    fn create(self:&, task:= Task) -> Result<Task, RepoError>  # mut self, move task
    fn update(self:&, task:& Task) -> Result<Task, RepoError>  # mut self, mut borrow task
    fn delete(self:&, id: Uuid) -> Result<(), RepoError>
    fn count_pending(self, user_id: Uuid) -> i64

# Production implementation
# Db is OWNED by the repo (moved in)
struct MausoleumTaskRepo
    db: Db                     # Owned, not a pointer!

fn mausoleum_task_repo_new(db:= Db) -> MausoleumTaskRepo
    MausoleumTaskRepo { db: db }

impl TaskRepository for MausoleumTaskRepo
    fn find(self, id: Uuid) -> Option<Task>
        self.db.query("tasks")
            .where_eq("id", id)
            .first::<Task>()
    
    fn find_by_user(self, user_id: Uuid) -> Vec<Task>
        self.db.query("tasks")
            .where_eq("user_id", user_id)
            .order_by("created_at", Desc)
            .all::<Task>()
    
    fn create(self:&, task:= Task) -> Result<Task, RepoError>
        # task ownership is moved in, we can mutate it
        var t = task
        t.id = uuid_new_v4()
        t.created_at = timestamp_now()
        t.updated_at = timestamp_now()
        self.db.insert("tasks", t)
    
    fn update(self:&, task:& Task) -> Result<Task, RepoError>
        task.updated_at = timestamp_now()
        self.db.update("tasks", task.id, task)

# Mock for testing - also owns its data
struct MockTaskRepo
    tasks: Vec<Task>
    next_error: Option<RepoError>

fn mock_task_repo_new() -> MockTaskRepo
    MockTaskRepo { 
        tasks: Vec.new(),
        next_error: None
    }
```

**Key ownership rules:**
- `self` = const borrow (can read)
- `self:&` = mutable borrow (can modify) 
- `db:= Db` = move ownership into struct
- `task:= Task` = move ownership into function
- `task:& Task` = mutable borrow

### 4. Services - Also Fixed

```ritz
# services/task_service.ritz

import spire.service { ServiceError, ValidationError, LimitError }
import repos.task_repo { TaskRepository }
import models { Task, User }

struct TaskService<R: TaskRepository>
    repo: R                    # Owned, not pointer!
    max_pending_tasks: i64

fn task_service_new<R: TaskRepository>(repo:= R) -> TaskService<R>
    TaskService::<R> {
        repo: repo,
        max_pending_tasks: 100
    }

impl<R: TaskRepository> TaskService<R>
    fn create_task(self:&, user: User, title: StrView) -> Result<Task, ServiceError>
        # Rule: title must be meaningful
        if title.len() < 3
            return Err(ValidationError("Title must be at least 3 characters"))
        
        if title.len() > 200
            return Err(ValidationError("Title must be at most 200 characters"))
        
        # Rule: max pending tasks per user
        let pending = self.repo.count_pending(user.id)
        if pending >= self.max_pending_tasks
            return Err(LimitError("Too many pending tasks (max 100)"))
        
        # Create the task
        let task = Task {
            id: Uuid.zero(),           # Will be set by repo
            title: title.to_string(),  # StrView -> String
            description: None,
            completed: false,
            created_at: Timestamp.zero(),
            updated_at: Timestamp.zero(),
            user_id: user.id
        }
        
        self.repo.create(task)  # Move task into repo
    
    fn complete_task(self:&, user: User, task_id: Uuid) -> Result<Task, ServiceError>
        match self.repo.find(task_id)
            None => Err(NotFoundError("Task not found"))
            Some(task) =>
                if task.user_id != user.id
                    return Err(ForbiddenError("Cannot modify another user's task"))
                
                if task.completed
                    return Err(ValidationError("Task is already completed"))
                
                task.completed = true
                self.repo.update(@&task)
    
    fn list_tasks(self, user: User) -> Vec<Task>
        self.repo.find_by_user(user.id)
```

### 7. Zeus Integration - App as a Module, Not a Binary

This is the crucial architectural point. Let me think through this properly:

```
┌───────────────────────────────────────────────────────────────┐
│                          VALET                                 │
│              (HTTP Server / Reverse Proxy)                     │
│        io_uring, TLS, static files, load balancing             │
│                              │                                 │
│                    ring buffer (shared memory)                 │
│                              │                                 │
├──────────────────────────────▼────────────────────────────────┤
│                          ZEUS                                  │
│               (App Server / Process Manager)                   │
│    - Loads app modules                                        │
│    - Manages worker processes                                  │
│    - Routes requests to handlers                               │
│    - Health checks, restarts, metrics                          │
│                              │                                 │
│                    (function calls, shared state)              │
│                              │                                 │
├──────────────────────────────▼────────────────────────────────┤
│                      YOUR SPIRE APP                            │
│                  (A module, not a binary)                      │
│                                                                │
│   ┌──────────────────────────────────────────────────────┐   │
│   │                   App Module                          │   │
│   │                                                       │   │
│   │   fn app_init(config: AppConfig) -> App               │   │
│   │   fn app_handle(app:&, req:= Request) -> Response     │   │
│   │   fn app_shutdown(app:= App)                          │   │
│   │                                                       │   │
│   │   # Internally wires:                                 │   │
│   │   # - Repositories (connected to Mausoleum/Tome)     │   │
│   │   # - Services                                        │   │
│   │   # - Presenters                                      │   │
│   │   # - Router                                          │   │
│   │                                                       │   │
│   └──────────────────────────────────────────────────────┘   │
└───────────────────────────────────────────────────────────────┘
```

**The App Contract:**

```ritz
# spire/app.ritz

# Apps implement this trait
trait SpireApp
    fn init(config: AppConfig) -> Self
    fn handle(self:&, req:= Request) -> Response
    fn shutdown(self:=)

# Zeus calls these entry points
# (similar to WSGI/ASGI in Python)
```

**Your App:**

```ritz
            cache: cache,
            router: router,
            task_service: task_service,
            user_service: user_service

        self.db.close()

[[spire_app]]
type App = MyApp
```

**The Flow:**

1. **Zeus starts** - Loads the app module, calls `App.init(config)`
2. **Valet receives HTTP** - Parses request, sends over ring buffer
3. **Zeus receives request** - Calls `app.handle(request)` on a worker
4. **Worker returns Response** - Zeus sends back to Valet
5. **Valet sends HTTP response** - Back to client

**Benefits:**
- App is a library, not a standalone binary
- Zeus manages lifecycle (restarts, scaling)
- Shared memory between Valet↔Zeus for zero-copy
- Hot reload possible (load new module, drain old workers)

Since the app is a module and routes hook into objects:

```ritz
# presenters/task_presenter.ritz

import spire.http { Request, Response, Router }
import spire.view { render }
import services.task_service { TaskService }
import views.tasks

struct TaskPresenter<R: TaskRepository>
    service: @TaskService<R>   # Reference to service (owned by App)

impl<R: TaskRepository> TaskPresenter<R>
    fn new(service: @TaskService<R>) -> TaskPresenter<R>
        TaskPresenter::<R> { service: service }
    
    fn register(self, router:& Router)
        # Routes reference self - the presenter is long-lived
        router.get("/tasks", self, Self::index)
        router.get("/tasks/new", self, Self::new_form)
        router.get("/tasks/:id", self, Self::show)
        router.post("/tasks", self, Self::create)
        router.post("/tasks/:id/complete", self, Self::complete)
    
    fn index(self, req: Request) -> Response
        let user = req.user()?
        let tasks = self.service.list_tasks(user)
        
        if req.wants_json()
            Response.ok().json(tasks)
        else
            Response.ok().html(render(tasks_index(tasks)))
    
    fn show(self, req: Request) -> Response
        let user = req.user()?
        let id = req.param::<Uuid>("id")?
        
        match self.service.get_task(user, id)
            Ok(task) =>
                if req.wants_json()
                    Response.ok().json(task)
                else
                    Response.ok().html(render(tasks_show(task)))
            Err(e) => e.to_response()
    
    fn create(self:&, req: Request) -> Response
        let user = req.user()?
        let body = req.json::<CreateTaskRequest>()?
        
        match self.service.create_task(user, body.title)
            Ok(task) =>
                if req.wants_json()
                    Response.created().json(task)
                else
                    Response.redirect("/tasks/{task.id}")
            Err(e) => e.to_response()
```
